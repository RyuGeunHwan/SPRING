# 쿠키와 세션 개념

```
1. 쿠키(웹(브라우저)에 저장하는 DB) : 
    1-1) 쿠키는 웹(브라우저)에 데이터를 저장
    1-2) 쿠키는 클라이언트의 상태 정보를 로컬에 저장했다가 참조한다.
        ex) *방문 사이트에서 로그인 시, "아이디와 비밀번호를 저장하시겠습니까?"
            *쇼핑몰의 장바구니 기능
            *자동로그인
            *팝업에서 "오늘 더 이상 이 창을 보지 않음" 체크

2. 세션(서버에 저장하는 임시DB) : 
    2-1) 세션은 서버에 데이터를 임시 저장
    2-2) 서버에서는 클라이언트를 구분하기 위해 *세션 ID를 부여*하며 웹 브라우저가 서버에 접속해서 브라우저를 종료할 때까지 인증상태를 유지한다.
    2-3) 접속 시간에 제한을 두어 일정 시간 응답이 없다면 정보가 유지되지 않게 설정이 가능
    2-4) 클라이언트에게 고유 세션 ID를 부여하므로써 보안이 쿠키보다 우수하지만 사용자가 많아지면 메모리를 많이 차지한다.
    ex) 보안 면에서 우수하기 때문에 로그인,개인정보 등 보안상 중요한 작업이 필요할 때 사용
```

## 쿠키와 세션을 사용하는 이유
```
1. HTTP 프로토콜의 특성이자 약점을 보완하기 위해서 쿠키 또는 세션을 사용.
2. 우선 HTTP의 특성으로 connectionless, stateless 두가지가 있다.
    2-1) connectionless : 클라이언트가 요청을 한 후 응답을 받으면 그 연결을 끊어 버리는 특징
         한마디로 클라이언트가  request하고, 서버가 response를 완료하면 접속을 끊는 특성이있다.
    2-2) stateless : 통신이 끝나면 상태를 유지하지 않는 특징
         ex) 쇼핑몰에서 로그인을 한뒤에 쿠키와 세션을 사용하지 않으면 페이지를 이동할 때마다 로그인을 해야한다.
             쿠키와 세션을 사용하면 사용자에 대한 인증을 유지하게 된다.

```
