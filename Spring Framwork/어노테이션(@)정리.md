## @Controller

```
1. url을 요청받는 곳이야~ 라고 알려주는 어노테이션!
2. @RestController :
    2-1) @RestController를 알기전 @ResponseBody : HTML의 Body를 가져오는것.
    2-2) @Controller는 @ResponseBody도 같이 사용하여야 한다.
    2-3) @RestController == @Controller + @ResponseBody 이다.
        즉, @RestController를 사용하면 @Controller와 @ResponseBody를 따로 사용하지 않아도 되는 것이다.
3. 위치 : url을 받는 @GetMapping을 감싸는 클래스에 @RestController 작성
```

## @Mapping 종류(다중맵핑 가능)

```
1. @GetMapping("/url주소") : GET 요청을 하는 API의 어노테이션.
    1-1) 데이터를 가져올 때 사용한다.
    1-2) 다중맵핑 가능 : 사용법 -> @GetMapping(value = {"/", "/index", "/main"})
    1-3) 1-2)풀이 : url주소에 index or main이 오면 아래 작성한 메소드를 실행할게~!

2. @PostMapping("/url주소") : POST 요청을 하는 API의 어노테이션
    데이터를 게시할 때 사용한다.

3. @PutMapping("/url주소") : PUT 요청을 하는 API의 어노테이션
    데이터를 수정할 때 사용한다.

4. @DeleteMapping("/url주소") : DELETE 요청을 하는 API의 어노테이션
    데이터를 삭제할 때 사용한다.

5. @PatchMapping("/url주소") : PATCH 요청을 하는 API의 어노테이션
    데이터를 수정할 때 사용한다.
```

## @AllArgsConstructor

```
위치
1. 생성자가 필요한 클래스에 작성
(@AllArgsConstructor을 사용하면 모든 필드변수가 생성자로 생기기 때문에
    원하는 매개변수의 생성자가 필요하다면 생성자를 따로 작성하여야한다.)

1. 생성자를 자동으로 만들고 생성자의 파라미터에 필드변수만큼 순서대로 자동으로 추가
ex)
@AllArgsConstructor
public class UserVO {
    private String name;
    private int age;
    private int count;
	private String addr;
}

class UserVO 안에 생성자가 선언 되어 있음.
public UserVO(String name, int age, int count, String addr){
    this.name = name;
    this.age = age;
    this.count = count;
    this.addr = addr;
}
```

## gradle lombok( == @Getter, @Setter or @Data)

```
위치
1. getter,setter가 필요한 클래스에 작성

생성방법
1. 구글 'spring lombok gradle' search!
2. 첫번째 피드 들어가서 'Gradle without a plugin'안에 있는

compileOnly 'org.projectlombok:lombok:1.18.24'
annotationProcessor 'org.projectlombok:lombok:1.18.24'
testCompileOnly 'org.projectlombok:lombok:1.18.24'
testAnnotationProcessor 'org.projectlombok:lombok:1.18.24'

복사

3. Spring(이클립스)에 들어가서 파일이름(코끼리 모양) : 'build.gradle'(빌드.그래들) 들어가

4. dependencies{
//(dependencies 중괄호 안에 붙여넣기)
//getter,setter 어노테이션으로 함수 생성

compileOnly 'org.projectlombok:lombok:1.18.24'
annotationProcessor 'org.projectlombok:lombok:1.18.24'
testCompileOnly 'org.projectlombok:lombok:1.18.24'
testAnnotationProcessor 'org.projectlombok:lombok:1.18.24'

 }

 5.  파일 'build.gradle' 오른쪽 마우스 클릭 후 하단에 Gradle -> Refresh... 클릭해서 새로고침 해준다.
 6. getter,setter 필요한 함수 위에 @Getter @Setter or @Data 작성

    ex) @Getter
        @Setter
        or
        @Data
        public class UserVO{
        private String name;
	    private int age;
        // 필드변수 name과 age에 대한 getter, setter 함수가 어노테이션 안에 내장된 것.
        }
```
